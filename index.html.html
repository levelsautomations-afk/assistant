Google Cloud — Gmail access
This lets the assistant read and send your email. Follow these steps:

1. Go to console.cloud.google.com and create a new project (name it anything).
2. In the left menu → APIs & Services → Enable APIs → search for Gmail API → Enable.
3. Go to APIs & Services → Credentials → Create Credentials → OAuth client ID.
4. Choose Web application. Under Authorized JavaScript origins, add your site URL (e.g. https://yourusername.github.io).
5. Copy the Client ID and paste it below.
6. Go to OAuth consent screen → add your email as a Test user.

4
Deploy to GitHub Pages (free hosting)
So you and your partner can both access it from any browser:

1. Go to github.com → create a free account → New repository → name it assistant → set to Public.
2. Upload this HTML file, renamed to index.html.
3. Go to repository Settings → Pages → Source: Deploy from a branch → select main → Save.
4. Your URL will be https://yourusername.github.io/assistant — share this with your partner.
5. Go back to your Google Cloud Console → Credentials → your OAuth client → add this URL to Authorized JavaScript origins.
Security note: Your API key is saved in your browser only (localStorage). Don't share the raw HTML file with the key already in it — each person enters their own key on first visit.
